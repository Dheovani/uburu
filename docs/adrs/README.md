# Architecture decision records

This directory records important architectural decisions that should remain understandable after the implementation has moved on.

ADRs are intentionally short. They should state the context, the decision, and the consequences. They are not a replacement for detailed design documents in `docs/`, but they preserve why a direction was chosen.
